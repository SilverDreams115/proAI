# proAI Architecture

## Vision

`proAI` is designed as a data-driven sports prediction platform, not as an
agent that improvises answers. The system clearly separates capture,
interpretation, modeling, and publication.

## Main layers

### 1. Sources

- structured team and match statistics
- schedules and historical results
- narrative context: news, injuries, suspensions, rotations
- odds or other market signals, if approved later

### 2. Ingestion

Each source is encapsulated in a connector. No connector should mix scraping,
normalization, and business logic in the same unit.

Responsibilities:

- download information
- identify the source and capture timestamp
- transform the output into the raw internal schema
- record capture errors and quality signals

### 3. Normalization

Convert names, competitions, dates, and events into canonical entities:

- `competition`
- `team`
- `match`
- `team_alias`
- `evidence_item`

### 4. Interpretation

Use language models only to extract signals from unstructured text:

- relevant injury
- possible rotation
- coaching change
- schedule fatigue
- competitive importance of the match

Every interpretation must produce summarized evidence and a confidence score.

### 5. Prediction

The engine produces `home / draw / away` probabilities based on:

- historical ratings
- recent form
- home/away strength
- goals scored and conceded
- interpreted context

The expected technical baseline is a mixture of:

- `Elo`
- `Poisson`
- supervised tabular model

### 6. Progol layer

Group the slate fixtures and return:

- per-match probabilities
- suggested selection
- confidence level
- primary evidence used
- auditable `Simple`, `Dobles`, and `Completa` ticket snapshots generated by the backend

### 7. Calibration

Production picks must be evaluated by competition, not only globally. The
walk-forward endpoint reports readiness per competition so the UI can keep
uncertain leagues covered instead of presenting them as safe.

## Design rules

- always persist evidence and source metadata
- never hide uncertainty
- support retraining and backtesting
- decouple web connectors from the prediction engine
