# Project instructions for OpenCode

- First inspect the project structure before making conclusions.
- Do not modify files unless explicitly asked.
- Prefer small, focused changes.
- Avoid reading generated or dependency directories such as node_modules, .next, dist, build, target, vendor, .venv, and .git.
- When asked to analyze, provide:
  1. project type
  2. important files
  3. risks
  4. suggested next steps
- When asked to change code, first provide a plan and wait for explicit approval.
